2  SQL Basics

Welcome to SQL Basics — the chapter where you learn to talk to databases, and databases (mostly) talk back.

SQL (Structured Query Language) is the universal language of data. It doesn’t matter if you’re working at a tiny startup or inside a corporation with more data than common sense — SQL is the tool that gets you to the numbers. Every data analyst uses it. Every data scientist knows it. Even Excel users secretly wish they had it.

To add a comment in SQL, use two dashes (--). SQL will politely ignore everything that follows — which is more than can be said for most group project feedback:

-- This is a comment. SQL ignores it. Unlike your assignment marker.

The basic building block of SQL is the query:

SELECT column1, column2, ...
FROM table_name
WHERE condition;
  • SELECT — what columns you want
  • FROM — which table to look in
  • WHERE — an optional filter (only return rows that match a condition)

SQL keywords are case-insensitive — select and SELECT both work — but writing them in uppercase is convention because we’re professionals (or at least pretending to be). Always end your statement with a semicolon (;). Think of it as the full stop at the end of your sentence. SQL insists.

2.1 The FreshMart Dataset

We’ll be working with two tables. First, the Customers table, which tracks FreshMart loyalty members:

2.2 Table Creation

It is not common for a data analyst to build database structures from scratch, but understanding the basics helps you read and work with existing databases confidently.

2.2.1 Creating a Table

Use CREATE TABLE to define a new table. You specify the table name, each column’s name, and its data type.

CREATE TABLE Customers (
    CustomerID     INT PRIMARY KEY,
    FirstName      VARCHAR(50),
    LastName       VARCHAR(50),
    Age            INT,
    Gender         CHAR(1),
    Suburb         VARCHAR(100),
    State          CHAR(3),
    MembershipTier VARCHAR(10),
    YearJoined     INT,
    TotalSpend     DECIMAL(10, 2),
    Churned        INT
);

INT stores whole numbers. VARCHAR(50) stores text up to 50 characters long. DECIMAL(10, 2) stores numbers with two decimal places — useful for money, because nobody wants to round Gary’s $520.00 to the nearest integer. PRIMARY KEY means this column uniquely identifies each row — no two customers can share a CustomerID.

2.2.2 Inserting Data

Use INSERT INTO to add a new row. Here is how FreshMart registered their most recent loyalty member:

INSERT INTO Customers (CustomerID, FirstName, LastName, Age, Gender, Suburb, State, MembershipTier, YearJoined, TotalSpend, Churned)
VALUES (7, 'Nina', 'Patel', 19, 'F', 'Brunswick', 'VIC', 'Bronze', 2024, 0.00, 0);

Nina has spent $0.00 so far. In her defence, she only signed up for the loyalty card because the checkout operator asked nicely.

2.2.3 Querying Data

Use SELECT to retrieve rows from a table. The asterisk * means “all columns”:

SELECT * FROM Customers;

This returns every column and every row in the table. Useful for a quick look, but in real analysis you will usually name the specific columns you need.

2.2.4 Updating Data

Use UPDATE to modify existing rows. Priya Sharma has just crossed into a new spending bracket:

UPDATE Customers
SET TotalSpend = 9000.00
WHERE CustomerID = 1;
Never forget the WHERE clause in UPDATE!

Without it, you will update every single row in the table. Every customer’s TotalSpend becomes $9,000.00. FreshMart’s finance team will not be pleased, and neither will your exam marker.

2.2.5 Deleting Data

Use DELETE to remove rows. Gary McSnarl has officially churned and asked to be removed from the loyalty database:

DELETE FROM Customers
WHERE CustomerID = 6;
Never forget the WHERE clause in DELETE either!

Without it, you delete every customer. Every single one. Marcus, Priya, even Nina who hasn’t bought anything yet — all gone. Always double-check your WHERE clause before running a delete.

You now know how to create and manage tables. But as a data analyst, you will spend almost all your time querying data that already exists. That is where the real action is.

2.3 SQL for Data Querying and Manipulation

2.3.1 Selecting Data

The SELECT statement retrieves data from a table. You list the columns you want, separated by commas:

SELECT column1, column2, ...
FROM table_name;

To see just the names and membership tiers of FreshMart’s customers — the useful columns, not the internal IDs:

SELECT FirstName, LastName, MembershipTier
FROM Customers;
FirstName LastName MembershipTier
Priya Sharma Gold
Jake Nguyen Bronze
Bev O’Brien Silver
Marcus Lee Gold
Amara Diallo Silver
Gary McSnarl Bronze
Nina Patel Bronze

To see just the product names and prices from the Products table:

SELECT ProductName, UnitPrice
FROM Products;

2.3.2 Filtering Data

The WHERE clause filters rows based on a condition. Only rows that satisfy the condition are returned — everything else is quietly discarded.

SELECT column1, column2, ...
FROM table_name
WHERE condition;

Show only customers who have spent more than $1,000 at FreshMart:

SELECT FirstName, LastName, TotalSpend
FROM Customers
WHERE TotalSpend > 1000;
FirstName LastName TotalSpend
Priya Sharma 8420.50
Bev O’Brien 3150.75
Marcus Lee 12300.00
Amara Diallo 1890.25

Jake and Nina did not make the cut. Jake is not surprised — he is 22. Nina is technically surprised because she forgot she had a loyalty card.

2.3.3 Sorting Data

Use ORDER BY to sort your results. ASC is ascending (A–Z, smallest–largest) and DESC is descending (Z–A, largest–smallest). The default is ascending.

Let’s rank customers from highest to lowest total spend:

SELECT FirstName, LastName, TotalSpend
FROM Customers
ORDER BY TotalSpend DESC;
FirstName LastName TotalSpend
Marcus Lee 12300.00
Priya Sharma 8420.50
Bev O’Brien 3150.75
Amara Diallo 1890.25
Gary McSnarl 520.00
Jake Nguyen 340.00
Nina Patel 0.00

Marcus is comfortably in first place. Nina is in last place. Nina is unbothered.

2.3.4 Distinct Values

The DISTINCT keyword returns only unique values — duplicates are removed. Useful when you want a list of categories, not a full row-by-row count.

Which product categories does FreshMart stock?

SELECT DISTINCT Category
FROM Products;

This returns each category name exactly once, even if multiple products share the same category (as is the case with Dairy and Beverages).

Which states are FreshMart customers from?

SELECT DISTINCT State
FROM Customers;

2.3.5 Multiple Conditions

You can combine conditions with logical operators: AND, OR, and NOT.

Customers who joined after 2020 and have spent more than $500 — your recent-but-engaged members:

SELECT FirstName, LastName, YearJoined, TotalSpend
FROM Customers
WHERE YearJoined > 2020 AND TotalSpend > 500;

Customers who are either churned or have spent nothing — potential re-engagement targets:

SELECT FirstName, LastName, TotalSpend, Churned
FROM Customers
WHERE Churned = 1 OR TotalSpend = 0;

Products that are not organic:

SELECT ProductName, IsOrganic
FROM Products
WHERE NOT IsOrganic = 1;

2.3.5.1 Exercises

  1. Write a query to select all customers who joined after 2020.
  2. Write a query to select distinct states from the Customers table.
  3. Write a query to select customers with a TotalSpend between $300 and $2,000.
  4. Write a query to find products that are organic and cost less than $7.00.
Solutions 
-- 1. Customers who joined after 2020
SELECT FirstName, LastName, YearJoined
FROM Customers
WHERE YearJoined > 2020;

-- 2. Distinct states
SELECT DISTINCT State
FROM Customers;

-- 3. Customers with TotalSpend between $300 and $2,000
SELECT FirstName, LastName, TotalSpend
FROM Customers
WHERE TotalSpend BETWEEN 300 AND 2000;

-- 4. Organic products under $7.00
SELECT ProductName, UnitPrice, IsOrganic
FROM Products
WHERE IsOrganic = 1 AND UnitPrice < 7.00;

2.3.6 Aggregating Data

Aggregate functions summarise many rows into a single number. These are the workhorses of data analysis — you will use them constantly.

Function What it does
COUNT() Counts the number of rows
SUM() Adds up all values
AVG() Calculates the average
MIN() Finds the smallest value
MAX() Finds the largest value

How many customers does FreshMart have on its loyalty database?

SELECT COUNT(*) AS TotalCustomers
FROM Customers;

The result is 7. Seven customers, one of whom has spent nothing and one of whom has spent enough to feed a village.

What is the total lifetime spend across all customers?

SELECT SUM(TotalSpend) AS TotalRevenue
FROM Customers;

What is the average unit price of products in the store?

SELECT AVG(UnitPrice) AS AvgUnitPrice
FROM Products;

What is the price range — cheapest to most expensive?

SELECT MIN(UnitPrice) AS CheapestProduct, MAX(UnitPrice) AS MostExpensiveProduct
FROM Products;

The Cola at $2.80 is the cheapest. The Almond Butter at $9.99 is the most expensive. Whether that is fair is a matter of personal philosophy.

2.3.6.1 Exercises

  1. Write a query to count the total number of products in the Products table.
  2. Write a query to find the average TotalSpend across all customers.
  3. Write a query to find the product with the highest UnitPrice.
  4. Write a query to find the total TotalSpend across all customers who have not churned.
  5. Write a query to find the earliest year a customer joined FreshMart.
  6. Write a query to find the minimum and maximum UnitPrice among organic products only.
  7. Write a query to count the number of customers who have spent more than $0.
Solutions 
-- 1. Total number of products
SELECT COUNT(*) AS TotalProducts
FROM Products;

-- 2. Average customer spend
SELECT AVG(TotalSpend) AS AvgTotalSpend
FROM Customers;

-- 3. Most expensive product
SELECT ProductName, UnitPrice
FROM Products
WHERE UnitPrice = (SELECT MAX(UnitPrice) FROM Products);

-- 4. Total spend among non-churned customers
SELECT SUM(TotalSpend) AS ActiveCustomerRevenue
FROM Customers
WHERE Churned = 0;

-- 5. Earliest year joined
SELECT MIN(YearJoined) AS EarliestJoinYear
FROM Customers;

-- 6. Price range for organic products
SELECT MIN(UnitPrice) AS CheapestOrganic, MAX(UnitPrice) AS MostExpensiveOrganic
FROM Products
WHERE IsOrganic = 1;

-- 7. Customers who have actually spent something
SELECT COUNT(*) AS SpendingCustomers
FROM Customers
WHERE TotalSpend > 0;

2.3.7 WHERE Clause (More Detail)

The WHERE clause supports a full range of comparison operators:

Operator Meaning
= Equal to
!= or <> Not equal to
> Greater than
< Less than
>= Greater than or equal to
<= Less than or equal to

Customers with a TotalSpend strictly greater than $1,000:

SELECT FirstName, LastName, TotalSpend
FROM Customers
WHERE TotalSpend > 1000;

Products whose price is not equal to $3.49 (i.e., not the Full Cream Milk):

SELECT ProductName, UnitPrice
FROM Products
WHERE UnitPrice != 3.49;

2.3.7.1 Exercises

  1. Write a query to find all products with a UnitPrice less than $5.00.
  2. Write a query to find customers who joined before 2022 and have spent more than $1,000.
  3. Write a query to find customers who are from QLD or have a TotalSpend of exactly $0.00.
Solutions 
-- 1. Budget-friendly products
SELECT ProductName, UnitPrice
FROM Products
WHERE UnitPrice < 5.00;

-- 2. Long-term, high-value customers
SELECT FirstName, LastName, YearJoined, TotalSpend
FROM Customers
WHERE YearJoined < 2022 AND TotalSpend > 1000;

-- 3. QLD customers or zero-spend members
SELECT FirstName, LastName, State, TotalSpend
FROM Customers
WHERE State = 'QLD' OR TotalSpend = 0.00;

2.3.8 GROUP BY Clause

GROUP BY groups rows that share the same value in a column, then lets you apply aggregate functions to each group separately. Think of it as sorting your data into buckets and doing maths on each bucket.

SELECT column1, aggregate_function(column2)
FROM table_name
GROUP BY column1;

How many customers does FreshMart have in each state?

SELECT State, COUNT(*) AS NumberOfCustomers
FROM Customers
GROUP BY State;

This groups the Customers rows by state and counts how many are in each. VIC has a strong showing.

State NumberOfCustomers
NSW 2
QLD 1
VIC 4

What is the average TotalSpend by membership tier?

SELECT MembershipTier, AVG(TotalSpend) AS AvgSpend
FROM Customers
GROUP BY MembershipTier;

This tells you, at a glance, how much each tier tends to spend — useful for understanding whether the loyalty tiers are doing their job.

2.3.8.1 Exercises

  1. Write a query to find the total TotalSpend grouped by State in the Customers table.
  2. Write a query to count the number of customers in each MembershipTier.
  3. Write a query to find the maximum TotalSpend by MembershipTier.
  4. Write a query to find the average UnitPrice by Category in the Products table.
  5. Write a query to count the number of products in each Category.
  6. Write a query to find the total TotalSpend for churned vs non-churned customers (group by Churned).
  7. Write a query to find the number of customers who joined in each year, grouped by YearJoined.
Solutions 
-- 1. Total spend by state
SELECT State, SUM(TotalSpend) AS TotalStateSpend
FROM Customers
GROUP BY State;

-- 2. Customer count by membership tier
SELECT MembershipTier, COUNT(*) AS CustomerCount
FROM Customers
GROUP BY MembershipTier;

-- 3. Highest spender by membership tier
SELECT MembershipTier, MAX(TotalSpend) AS MaxSpend
FROM Customers
GROUP BY MembershipTier;

-- 4. Average unit price by product category
SELECT Category, AVG(UnitPrice) AS AvgPrice
FROM Products
GROUP BY Category;

-- 5. Number of products per category
SELECT Category, COUNT(*) AS ProductCount
FROM Products
GROUP BY Category;

-- 6. Total spend by churn status
SELECT Churned, SUM(TotalSpend) AS TotalSpend
FROM Customers
GROUP BY Churned;

-- 7. Customers joining each year
SELECT YearJoined, COUNT(*) AS NewMembers
FROM Customers
GROUP BY YearJoined;

2.3.9 HAVING Clause

HAVING is like WHERE for groups — it filters the results after GROUP BY has been applied. Use it when your condition involves an aggregate function like COUNT() or AVG().

SELECT column1, aggregate_function(column2)
FROM table_name
GROUP BY column1
HAVING condition;

Find membership tiers where the average TotalSpend exceeds $3,000:

SELECT MembershipTier, AVG(TotalSpend) AS AvgSpend
FROM Customers
GROUP BY MembershipTier
HAVING AVG(TotalSpend) > 3000;

Only the Gold tier qualifies — which is exactly what you’d hope for from your Gold members.

Find product categories where the average unit price is above $6.00:

SELECT Category, AVG(UnitPrice) AS AvgPrice
FROM Products
GROUP BY Category
HAVING AVG(UnitPrice) > 6.00;
WHERE vs HAVING — Know the Difference
  • WHERE filters individual rows before grouping. Use it on regular column values.
  • HAVING filters groups after grouping. Use it on the result of aggregate functions.

This query will throw an error because you cannot use an aggregate function inside WHERE:

-- ❌ This does NOT work
SELECT MembershipTier, AVG(TotalSpend)
FROM Customers
WHERE AVG(TotalSpend) > 3000
GROUP BY MembershipTier;

This is the correct version:

-- ✅ This works
SELECT MembershipTier, AVG(TotalSpend)
FROM Customers
GROUP BY MembershipTier
HAVING AVG(TotalSpend) > 3000;

2.3.9.1 Exercises

  1. Write a query to find states where the total TotalSpend of all customers exceeds $5,000.
  2. Write a query to find product categories that contain more than 1 product.
Solutions 
-- 1. States with combined customer spend above $5,000
SELECT State, SUM(TotalSpend) AS TotalStateSpend
FROM Customers
GROUP BY State
HAVING SUM(TotalSpend) > 5000;

-- 2. Categories with more than one product
SELECT Category, COUNT(*) AS ProductCount
FROM Products
GROUP BY Category
HAVING COUNT(*) > 1;

2.3.10 NULL Values

NULL in SQL means unknown or missing. It is not zero, it is not an empty string, and it is not false. It is the database equivalent of a friend saying “I’ll let you know” and then never letting you know.

Note that Nina Patel’s TotalSpend is 0.00, not NULL — she has a recorded value of zero. A NULL would mean the data is entirely absent, which is a different situation. In a larger database, you might encounter NULL in columns like LastName (for a customer who only provided a first name) or State (if the address was not captured).

You cannot compare NULL with =. This will silently return nothing:

-- ❌ Does NOT work as expected
SELECT FirstName FROM Customers WHERE State = NULL;

Instead, use IS NULL or IS NOT NULL:

-- Customers with no state on record
SELECT FirstName, LastName, State
FROM Customers
WHERE State IS NULL;

-- Customers who definitely have a state recorded
SELECT FirstName, LastName, State
FROM Customers
WHERE State IS NOT NULL;

2.3.10.1 Exercises

  1. Write a query to find all products whose Brand is NULL.
  2. Write a query to find all customers whose Suburb is not NULL.
Solutions 
-- 1. Products with no brand on file
SELECT ProductName
FROM Products
WHERE Brand IS NULL;

-- 2. Customers with a recorded suburb
SELECT FirstName, LastName, Suburb
FROM Customers
WHERE Suburb IS NOT NULL;

2.3.11 LIMIT Clause

The LIMIT clause caps the number of rows returned. Useful when you want a quick peek at a large table, or when you are hunting for the top or bottom performers.

SELECT *
FROM Products
LIMIT 3;

This returns only the first 3 rows. To find the 3 most expensive products:

SELECT ProductName, UnitPrice
FROM Products
ORDER BY UnitPrice DESC
LIMIT 3;

That would be Almond Butter ($9.99), Chicken Breast ($8.50), and Free Range Eggs ($7.99). Healthy eating is not cheap.

SQL Dialect Note

LIMIT works in MySQL, PostgreSQL, and SQLite. In Microsoft SQL Server, use SELECT TOP n instead:

SELECT TOP 3 ProductName, UnitPrice
FROM Products
ORDER BY UnitPrice DESC;

When in doubt, ask your AI assistant which variant your database system uses.

2.3.11.1 Exercises

  1. Write a query to find the 3 customers with the highest TotalSpend.
  2. Write a query to find the cheapest product in the store — return just 1 row.
Solutions 
-- 1. Top 3 spenders
SELECT FirstName, LastName, TotalSpend
FROM Customers
ORDER BY TotalSpend DESC
LIMIT 3;

-- 2. The cheapest product
SELECT ProductName, UnitPrice
FROM Products
ORDER BY UnitPrice ASC
LIMIT 1;
-- That's the Cola at $2.80. Jake approves.

2.3.12 LIKE Operator

LIKE lets you search for text patterns inside a column. It is used inside the WHERE clause and works with two wildcards:

  • % — matches any sequence of characters (including no characters at all)
  • _ — matches exactly one character

Products whose name starts with “Organic”:

SELECT ProductName
FROM Products
WHERE ProductName LIKE 'Organic%';

This returns Organic Oats 500g. Clean eating, found efficiently.

Products whose name contains the word “Butter”:

SELECT ProductName, UnitPrice
FROM Products
WHERE ProductName LIKE '%Butter%';

Customers whose last name starts with “P”:

SELECT FirstName, LastName
FROM Customers
WHERE LastName LIKE 'P%';

2.3.12.1 Exercises

  1. Write a query to find all products whose ProductName contains the word “Chocolate”.
  2. Write a query to find products whose ProductName ends with “6pk”.
  3. Write a query to find customers whose FirstName starts with “A”.
Solutions 
-- 1. Chocolate products (there is one, and it is important)
SELECT ProductName, UnitPrice
FROM Products
WHERE ProductName LIKE '%Chocolate%';

-- 2. Products sold in six-packs
SELECT ProductName, UnitPrice
FROM Products
WHERE ProductName LIKE '%6pk';

-- 3. Customers whose first name starts with A
SELECT FirstName, LastName
FROM Customers
WHERE FirstName LIKE 'A%';
-- That's Amara Diallo.

2.3.13 IN Operator

The IN operator checks if a value matches any value in a list. It is a clean alternative to chaining multiple OR conditions.

SELECT column1, column2
FROM table_name
WHERE column_name IN (value1, value2, value3);

Find customers who are Gold or Silver members:

SELECT FirstName, LastName, MembershipTier
FROM Customers
WHERE MembershipTier IN ('Gold', 'Silver');

Much neater than writing:

WHERE MembershipTier = 'Gold' OR MembershipTier = 'Silver'

You can also use NOT IN to exclude specific values:

-- Products NOT in the Dairy or Beverages categories
SELECT ProductName, Category
FROM Products
WHERE Category NOT IN ('Dairy', 'Beverages');

2.3.13.1 Exercises

  1. Write a query to find customers from VIC, NSW, or QLD.
  2. Write a query to find products that are not in the Bakery or Snacks categories.
Solutions 
-- 1. Customers from major eastern states
SELECT FirstName, LastName, State
FROM Customers
WHERE State IN ('VIC', 'NSW', 'QLD');

-- 2. Products outside Bakery and Snacks
SELECT ProductName, Category
FROM Products
WHERE Category NOT IN ('Bakery', 'Snacks');

2.3.14 BETWEEN Operator

BETWEEN filters for values within a range. It is inclusive — both the lower and upper bounds are included in the results. It works with numbers, dates, and text.

SELECT column1, column2
FROM table_name
WHERE column_name BETWEEN value1 AND value2;

Products priced between $3.00 and $7.00:

SELECT ProductName, UnitPrice
FROM Products
WHERE UnitPrice BETWEEN 3.00 AND 7.00;

Customers who joined between 2019 and 2022 (inclusive):

SELECT FirstName, LastName, YearJoined
FROM Customers
WHERE YearJoined BETWEEN 2019 AND 2022;

2.3.14.1 Exercises

  1. Write a query to find products with a UnitPrice between $5.00 and $9.00.
  2. Write a query to find customers with a TotalSpend between $1,000 and $10,000.
Solutions 
-- 1. Mid-range products
SELECT ProductName, UnitPrice
FROM Products
WHERE UnitPrice BETWEEN 5.00 AND 9.00;

-- 2. Mid-range spenders
SELECT FirstName, LastName, TotalSpend
FROM Customers
WHERE TotalSpend BETWEEN 1000 AND 10000;

2.3.15 Aliases with AS

Aliases give a column or table a temporary name in your query results. They do not change the actual database — they just make the output easier to read and share.

SELECT column_name AS alias_name
FROM table_name AS table_alias;

Let’s produce a readable customer report instead of raw column names:

SELECT FirstName AS First, LastName AS Last, TotalSpend AS LifetimeValue, MembershipTier AS Tier
FROM Customers AS C;

Aliases are especially useful with aggregate functions, which otherwise produce awkward output names like COUNT(*) or AVG(TotalSpend):

SELECT
    COUNT(*) AS TotalCustomers,
    AVG(TotalSpend) AS AvgLifetimeValue,
    SUM(TotalSpend) AS TotalRevenue
FROM Customers;

Without aliases, your column header would literally say COUNT(*). Not a great look in a client dashboard.

2.3.15.1 Exercises

  1. Write a query to select ProductName and UnitPrice from Products, renaming them as Product and Price.
  2. Write a query that counts all products and calculates the average UnitPrice, with clear aliases on both output columns.
Solutions 
-- 1. Rename product columns
SELECT ProductName AS Product, UnitPrice AS Price
FROM Products;

-- 2. Product summary with clean aliases
SELECT
    COUNT(*) AS TotalProducts,
    AVG(UnitPrice) AS AvgPrice
FROM Products;
