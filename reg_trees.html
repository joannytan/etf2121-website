Regression Trees

The linear regression in Week 7 gave us an equation. This week we get something the marketing team can pin on the wall: a flowchart of yes/no questions leading to a spend prediction. Decision trees trade mathematical precision for human interpretability — and that trade is often worth making.

11.1 9.1 How a Decision Tree Thinks

A decision tree makes predictions by asking a series of yes/no questions about the data. It’s the model equivalent of the flowcharts you might draw on a whiteboard.

Here’s a simple example of what a FreshMart spending tree might look like:

Is TotalTransactions > 12?
  ├─ Yes → Is AvgTransactionValue > $80?
  │    ├─ Yes → Predict: $1,450
  │    └─ No   → Predict: $640
  └─ No   → Is DaysSinceFirstPurchase > 400?
       ├─ Yes → Predict: $195
       └─ No   → Predict: $72

Each path from the top to a leaf gives the model’s prediction for that group of customers. The prediction at each leaf is the average TotalSpend of the training customers who fell into that group.

11.1.1 How does the tree choose where to split?

At each node, the tree tries every possible split on every predictor and picks the one that most reduces the variance of the outcome within each resulting group. Splitting on “TotalTransactions > 12” is useful if it separates high-spending from low-spending customers more cleanly than any other cut. The goal is to create groups (leaves) where customers are as similar to each other as possible in terms of spend.

Important✏️ Concept Check — 9.1

Q1. In a regression tree, what value does the model predict at each leaf?

Q2. A tree splits on “TotalTransactions > 5.” After the split, one group has very similar spend values and the other has very diverse spend values. Was this a good split? Why?

Q3. What is one advantage a decision tree has over a linear regression equation, from a business communication standpoint?

Tip✅ Answers

A1. The mean of the outcome (TotalSpend) for all training observations that fell into that leaf.

A2. Not ideal — we’d prefer both groups to have low variance after the split (each group’s customers are similar to each other). A good split creates two homogeneous groups. If one group is still very diverse, the tree will need more splits to capture it.

A3. A decision tree produces a set of explicit if-then rules that any manager can read and understand without statistical training. An equation like TotalSpend = 25 + 18×Transactions + 0.4×Days... requires interpretation; a tree flowchart is self-explanatory.

Q1. tree_depth = 4 means the tree can ask at most 4 questions in sequence. What does tree_depth = 1 produce?

Q2. min_n = 10 means a node must have at least 10 observations to split further. What would happen if we set min_n = 1?

Q3. Compare the top split in your tree to the strongest predictor from the linear regression (Chapter 7). Are they the same variable?

Tip✅ Answers

A1. A single yes/no question — the simplest possible tree with just two leaves. Sometimes called a “stump.”

A2. The tree would keep splitting until every leaf contained only a single observation. It would perfectly predict the training data but would be terrible on new data — pure overfitting.

A3. Typically yes — the variable the tree first splits on is the most powerful predictor of spend, just as the largest coefficient in linear regression indicates the most important predictor. Both methods tend to agree on which variables matter most.

11.3 9.3 Overfitting and Tree Depth

A deep tree can memorise the training data almost perfectly — but fail badly on new data. This is called overfitting: the model has learned the noise in the training data, not the true underlying pattern.

We can see this by comparing training vs test RMSE across different tree depths.

What to look for:

  • As depth increases, training RMSE always falls (the tree memorises more detail)
  • Test RMSE falls at first, then levels off or rises — this is where overfitting begins
  • The “sweet spot” is the depth where test RMSE is lowest before it starts increasing
Important✏️ Concept Check — 9.3

Q1. At depth = 10, training RMSE is much lower than test RMSE. What is happening?

Q2. At depth = 1, both training and test RMSE are high. What does this mean?

Q3. How is the overfitting problem in trees similar to the concept of using the test set (not the training set) for evaluation that we discussed in Chapter 7?

Tip✅ Answers

A1. The tree has memorised the training data — it knows exactly which leaf each training customer lands in. But those very specific rules don’t generalise; new customers don’t match those precise patterns, so test errors are much larger. This is overfitting.

A2. A depth-1 tree asks only one question. It’s too simple to capture the real pattern in the data — both training and test errors are high. This is underfitting — the opposite problem.

A3. Both reflect the same core idea: evaluating a model on the data it was trained on gives an overly optimistic picture. We always need to test on held-out data to see how the model truly performs on new, unseen observations.

11.4 9.4 Trees vs Linear Regression

Now let’s compare our best decision tree directly against the linear regression model from Chapter 7, using the same test set.

Key difference in predictions: The tree produces predictions that cluster at distinct values (the leaf averages), creating horizontal bands in the actual-vs-predicted plot. Linear regression produces a smooth spread of predictions. Trees tend to struggle more with very high-spend outliers like Marcus Lee — they can only predict within the range of their training leaf averages.

Important✏️ Concept Check — 9.4

Q1. In the actual-vs-predicted plot, why does the decision tree show horizontal “stripes” while linear regression shows a smoother diagonal?

Q2. Linear regression gives slightly better RMSE. Does this mean we should always use linear regression? What might favour choosing a tree?

Q3. Name one type of FreshMart business audience that would prefer seeing a decision tree over a regression equation.

Tip✅ Answers

A1. Because a tree can only predict the average spend of the training customers in each leaf — there are a limited number of distinct prediction values (one per leaf). Linear regression can output a unique value for every set of inputs.

A2. Not necessarily. A tree is far easier for non-technical stakeholders to understand and audit. If FreshMart needs to explain to store managers why a customer is being targeted, a tree’s rules are much clearer than a regression equation. Interpretability has real business value.

A3. Store managers, field sales teams, or customer service staff who need to apply rules manually would strongly prefer a tree. They can follow the flowchart without understanding regression.

NoteExercise 9.2 — Depth Tuning

From the overfitting plot in Section 9.3, identify the optimal tree depth (lowest test RMSE). Fit a final tree at that depth and report its test RMSE and R².

Tip✅ Solution 
# Identify best depth from the depth_results tibble
best_depth <- depth_results |>
  filter(set == "Test") |>
  slice_min(.estimate) |>
  pull(depth)

cat("Optimal depth:", best_depth, "\n")

# Fit final tree at optimal depth
final_tree <- workflow() |>
  add_recipe(tree_recipe) |>
  add_model(decision_tree(tree_depth = best_depth, min_n = 10) |>
              set_engine("rpart") |> set_mode("regression")) |>
  fit(data = fm_train)

augment(final_tree, new_data = fm_test) |>
  metrics(truth = TotalSpend, estimate = .pred) |>
  filter(.metric %in% c("rmse", "rsq"))

NoteExercise 9.4 — When to Use Each Model

For each scenario below, decide whether you’d recommend a linear regression or a decision tree, and briefly justify your choice:

  1. FreshMart wants to predict customer lifetime spend for internal forecasting. The predictions will be used in a financial spreadsheet.
  2. FreshMart’s store managers want to understand which customers to approach for a loyalty upgrade. The rules need to be explainable in a 30-second conversation.
  3. FreshMart is building an automated email targeting system where the model runs 10,000 predictions per day with no human review.
Tip✅ Sample Answers
  1. Linear regression — the predictions are continuous and the finance team cares about accuracy. R² and RMSE are familiar metrics in that context.
  2. Decision tree — store managers need rules they can explain to customers. “We’re reaching out because you’ve visited more than 10 times and your average spend is over $50” is a clear tree-derived rule.
  3. Either — in an automated system, interpretability matters less. Choose based on which has lower test RMSE. If both are similar, the linear regression may be preferable for its speed and simplicity.
